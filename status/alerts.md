# 问题预警机制

**版本**: v1.0
**创建日期**: 2026-03-02

---

## 预警分级

| 级别 | 含义 | 触发条件 | 响应时间 |
|------|------|----------|----------|
| 🔴 P0 | 紧急 | 硬件故障、数据丢失、安全风险 | 立即 |
| 🟠 P1 | 重要 | 进度延迟、关键技术问题 | 当天 |
| 🟡 P2 | 一般 | 计划偏差、资源不足 | 本周 |
| 🔵 P3 | 提示 | 优化机会、知识更新 | 定期 |

---

## 预警类型

### 技术类

| 预警类型 | 检查项 | 级别 |
|----------|--------|------|
| 代码问题 | 编译错误、测试失败 | P1 |
| 依赖问题 | 包版本冲突、缺失 | P1 |
| 性能问题 | 响应慢、内存高 | P2 |
| 安全问题 | 敏感信息泄露 | P0 |

### 项目类

| 预警类型 | 检查项 | 级别 |
|----------|--------|------|
| 进度延迟 | 任务逾期超过2天 | P1 |
| 阻塞 | 等待用户决策超过1天 | P1 |
| 资源 | 缺少关键数据/文档 | P2 |
| 依赖 | 等待其他项目 | P2 |

### 知识类

| 预警类型 | 检查项 | 级别 |
|----------|--------|------|
| 知识缺口 | 遇到无法回答的问题 | P2 |
| 知识过期 | 发现错误信息 | P2 |
| 技术更新 | 发现新技术/新方案 | P3 |

---

## 预警格式

```markdown
## [日期] 预警 #[N]

### 级别
🔴 P0 / 🟠 P1 / 🟡 P2 / 🔵 P3

### 类型
[技术/项目/知识]

### 描述
[问题描述]

### 影响
[可能的影响]

### 建议
[建议的解决方案]

### 状态
- [ ] 待处理
- [ ] 处理中
- [ ] 已解决
- [ ] 已忽略
```

## 检查清单

### 每日检查 (心跳)

- [ ] GitHub新commit？
- [ ] 任务进度正常？
- [ ] 有阻塞问题？

### 每周检查

- [ ] 计划vs实际偏差？
- [ ] 知识需要更新？
- [ ] 有优化机会？

### 每月检查

- [ ] 整体进度？
- [ ] 技术栈过时？
- [ ] 目标需要调整？

## 响应流程

```
检测到问题
    ↓
判断级别
    ↓
P0 → 立即通知 + 等待响应
P1 → 当天报告 + 处理
P2 → 本周报告 + 计划处理
P3 → 定期报告 + 记录
```
